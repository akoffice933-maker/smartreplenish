## 🎯 Кратко о проекте

**SmartReplenish** — это промышленная платформа для автоматического прогнозирования спроса и оптимизации товарных запасов розничных сетей. Система использует машинное обучение и математическую оптимизацию для снижения издержек и увеличения доступности товаров.

### 💰 Ценностное предложение

| Проблема | Решение SmartReplenish |
|----------|------------------------|
| Избыточные запасы (замороженные деньги) | **Снижение на 8-18%** |
| Дефицит товаров (потеря продаж) | **OSA ≥ 97%** |
| Низкая точность прогнозов | **WAPE < 22%** (горизонт 7 дней) |
| Ручное планирование заказов | **Полная автоматизация** |
| Списания просрочки | **Снижение на 15-30%** |

## 🚀 Ключевые возможности

### 1. Прогнозирование спроса (Forecasting)

- **Ансамбль моделей**: LightGBM + CatBoost для максимальной точности
- **Горизонт прогноза**: 1-30 дней
- **Учёт факторов**: сезонность, промо, цены, календарь
- **Автоматическое переобучение**: ежедневное обновление моделей
- **Backtesting**: валидация на исторических данных

```python
# Пример запроса прогноза
import requests

response = requests.post(
    "http://api.smartreplenish.com/api/v1/forecast",
    json={
        "shop_id": "STORE-001",
        "sku_id": "SKU-12345",
        "horizon_days": 7
    }
)
forecast = response.json()
# → predictions: [{date, value, lower_bound, upper_bound}, ...]
```

### 2. Оптимизация заказов (Optimization)

- **Математическая оптимизация**: линейное программирование (OR-Tools)
- **Ограничения**: мин. партия, кратность, вместимость склада
- **Страховой запас**: динамический расчёт по variability спроса
- **Срок годности**: учёт для fresh-продуктов

```python
# Пример расчёта заказа
response = requests.post(
    "http://api.smartreplenish.com/api/v1/optimization",
    json={"shop_id": "STORE-001", "horizon_days": 7}
)
order = response.json()
# → recommendations: [{sku_id, recommended_quantity, safety_stock}, ...]
```

### 3. Алерты и уведомления (Alerting)

| Тип алерта | Описание | Действие |
|------------|----------|----------|
| **Stockout Risk** | Риск дефицита (остаток < прогноз) | Срочный заказ |
| **Overstock** | Избыток (остаток > 2× прогноз) | Отмена заказа, промо |
| **Expiry** | Истекает срок годности | Распродажа, списание |
| **Forecast Drift** | Отклонение прогноза > 20% | Переобучение модели |

### 4. Feature Store

- **Лаги**: sales_lag_7d, sales_lag_14d, sales_lag_28d
- **Rolling статистики**: mean_7d, std_7d, trend_14d
- **Календарные признаки**: day_of_week, month, is_holiday
- **Промо признаки**: is_promo, promo_depth, promo_duration

### 5. Dashboard и отчётность

- **KPI мониторинг**: WAPE, OSA, Inventory Turnover
- **Grafana дашборды**: 10+ готовых визуализаций
- **Экспорт данных**: CSV, Excel, API

## 🛠 Быстрый старт

### Предварительные требования

- Docker 24.0+ и Docker Compose 2.20+
- Python 3.11+ (для локальной разработки)
- 8 GB RAM, 4 CPU cores (рекомендуемые)

### Установка за 5 минут

```bash
# 1. Клонирование репозитория
git clone https://github.com/akoffice933-maker/smartreplenish.git
cd smartreplenish

# 2. Настройка окружения
cp .env.example .env

# 3. Запуск всех сервисов
docker-compose up -d

# 4. Проверка работоспособности
curl http://localhost:8080/health
# → {"status": "healthy", "version": "1.0.0"}
```

## 📊 Мониторинг и Observability

### Метрики (Prometheus)

- `http_requests_total`: Количество запросов по сервисам
- `http_request_duration_seconds`: Время обработки (p50, p95, p99)
- `forecast_wape`: Точность прогнозов (скользящее среднее)
- `model_inference_latency`: Время инференса ML-моделей
- `order_recommendation_count`: Количество сгенерированных рекомендаций

### Дашборды (Grafana)

1. **System Health**: Статус сервисов, latency, error rate
2. **Forecast Accuracy**: WAPE/MAPE/SMAPE по категориям
3. **Business KPIs**: OSA, Inventory Turnover, Stockout Rate
4. **Resource Usage**: CPU, память, диск, сеть

### Алерты

- Service Down (Prometheus → Telegram/Email)
- Forecast WAPE > 25% (автоматическое переобучение)
- Kafka Lag > 10000 (проблемы с обработкой)
